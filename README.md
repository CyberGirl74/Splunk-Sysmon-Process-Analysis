# Splunk Sysmon Process Analysis

## Overview

This project demonstrates how I used Splunk Enterprise 10.4 and Microsoft Sysmon to analyze Windows process creation events using Sysmon Event ID 1. The purpose of this lab was to gain hands-on experience writing Splunk Search Processing Language (SPL) queries, investigate parent-child process relationships, establish a baseline of normal Windows activity, and strengthen foundational blue team investigation skills.

---

## Objectives

* Configure Splunk to ingest Sysmon logs.
* Analyze Windows Event ID 1 (Process Creation).
* Practice writing SPL queries.
* Investigate parent-child process relationships.
* Identify frequently executed Windows processes.
* Develop practical blue team investigation skills.

---

## Tools Used

* Splunk Enterprise 10.4
* Microsoft Sysmon
* Windows 11
* Windows Event Logs
* Splunk Search Processing Language (SPL)

# Investigation 1 – Parent Process Analysis for PowerShell

## Purpose

PowerShell is frequently abused by attackers to execute scripts and malicious commands. This query identifies which parent processes launched PowerShell so that analysts can distinguish expected activity from suspicious behavior.

## SPL Query

```spl
index=* EventCode=1
Image="*powershell.exe"
| stats count by ParentImage
| sort -count
```

## Results

The investigation showed that nearly all PowerShell executions originated from **splunkd.exe**, indicating that Splunk launched PowerShell during its normal indexing and monitoring operations.

Additional parent processes included:

* WindowsTerminal.exe
* explorer.exe

These parent processes are expected and represent legitimate Windows activity.

# Investigation 2 – Top 10 Process Creation Events

## Purpose

This query identifies the ten most frequently executed processes captured by Sysmon Event ID 1. Establishing a baseline of normal activity helps security analysts recognize abnormal process execution during future investigations.

## SPL Query

```spl
index=* EventCode=1
| top Image limit=10
```

## Results

The results revealed that the majority of process creation events were generated by legitimate Splunk background services.

Frequently observed processes included:

* splunk-optimize.exe
* splunk-powershell.exe
* python3.9.exe
* cmd.exe
* splunkd.exe
* postgres.exe
* splunk-regmon.exe
* splunk-netmon.exe
* splunk-admon.exe
* splunk-MonitorNoHandle.exe

These are legitimate Splunk Enterprise components responsible for indexing, monitoring, database management, and system operations. Understanding expected background processes is essential because security analysts must first establish a baseline of normal behavior before identifying anomalies.
